## Overview

Query the stablecoin balance of an internal (custodial) address. This endpoint requires both `transfer_type` and `value_type` because a single custodial address can hold multiple tokens across multiple chains.

:::note
Balances are only available for `type=internal` custodial addresses managed by Brale.
:::

## Endpoint

`GET /accounts/{account_id}/addresses/{address_id}/balance`

### Required query parameters

| Parameter        | Description                     | Example            |
| ---------------- | -------------------------------- | ------------------ |
| `transfer_type`  | The blockchain network          | `solana`, `base`, `ethereum` |
| `value_type`     | The token symbol (case-sensitive) | `SBC`, `USDC`, `PYUSD` |

## Example request

```bash
curl --request GET \
  --url "https://api.brale.xyz/accounts/${ACCOUNT_ID}/addresses/${ADDRESS_ID}/balance?transfer_type=base&value_type=SBC" \
  --header "Authorization: Bearer ${AUTH_TOKEN}"
```

## Example response

```json
{
  "address": {
    "id": "2VcUIonJeVQzFoBuC7LdFT0dRe4",
    "address": "0xcdEA458750b9A8D6C4Ba8B3D68CE98Ba2330352A"
  },
  "balance": {
    "value": "45314.07",
    "currency": "USD"
  },
  "value_type": "SBC",
  "transfer_type": "base"
}
```

## Notes

- Works only for internal custodial addresses (`type=internal`).
- Always send both `transfer_type` and `value_type`; a custodial address can hold multiple tokens across chains.
- See [Value Types](/coverage/value-types) and [Transfer Types](/docs/coverage/transfer-types) for supported combinations.
